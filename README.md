# نظام نقاط البيع — مقهى/مطعم

نظام POS متكامل مع محاسبة مزدوجة، إدارة مخزون، وتقارير تفصيلية.

**واجهة المستخدم:** التصميم والتخطيط **مخصّصان لسطح المكتب** (كمبيوتر كاشير / متصفح بعرض كافٍ). لا يُفترض تشغيل الواجهات على الهاتف كهدف رئيسي؛ يُفضّل استغلال المساحة الأفقية والعمودية لتقليل التنقّل بين الصفحات. **المظهر الموحّد:** لوحة ألوان الكاشير الكلاسيكي (أزرق داكن `#1e3a5f`، خلفية `#e8eaed`) عبر `classic-theme.css` و`base.html` / `shell/base.html`.

---

## التثبيت بأمر واحد

```bash
git clone https://github.com/nobodycp/cafe-pos.git && cd cafe-pos && bash install.sh
```

التثبيت يعمل تلقائياً:
- تحميل الكود وتثبيت المكتبات
- إعداد قاعدة البيانات (فارغة — تعبيها أنت)
- إنشاء حساب مدير بكلمة مرور تختارها
- تشغيل النظام كخدمة (يرجع يشتغل تلقائياً لو السيرفر أعاد تشغيل)

### المتطلبات
- سيرفر Linux (Ubuntu/Debian مفضّل)
- Python 3.8+
- Git

## بعد التثبيت

افتح المتصفح على:

```
http://عنوان-السيرفر:8000
```

سجّل دخول بحساب المدير الذي أنشأته أثناء التثبيت.

---

## المتصفح يعطي «This site can’t be reached» أو timeout

الخدمة قد تكون شغّالة على السيرفر لكن **المنفذ مقفول من الإنترنت**.

1. **لوحة مزوّد السحابة** (Hetzner Cloud Firewall، DigitalOcean، AWS Security Group، إلخ):  
   أضف قاعدة **Inbound** — بروتوكول **TCP** — المنفذ **8000** (أو المنفذ الذي اخترته) — المصدر **0.0.0.0/0** (أو IP مكتبك فقط لزيادة الأمان).

2. **UFW على Ubuntu** (إن كان مفعّلاً):

   **مهم:** SSH يعمل على المنفذ **22**. لا تفعّل `ufw enable` ولا تضيف قواعد بدون السماح بـ SSH أولاً وإلا قد تُقفل الدخول للسيرفر.

   ```bash
   sudo ufw allow OpenSSH      # الأفضل على Ubuntu (يشمل SSH)
   # أو إن لم يتوفر الاسم أعلاه:
   sudo ufw allow 22/tcp

   sudo ufw allow 8000/tcp     # أو المنفذ الذي اخترته للتطبيق
   sudo ufw reload
   sudo ufw status
   ```

   ثم فقط إذا كنت متأكداً من القواعد: `sudo ufw enable`

3. **اختبار من داخل السيرفر** (لو رجع رقم وليس خطأ، التطبيق يعمل):

   ```bash
   curl -sS -o /dev/null -w "%{http_code}\n" http://127.0.0.1:8000/
   ```

---

## الصفحة تفتح لكن «400 Bad Request» أو تسجيل الدخول / CSRF يفشل

هذا **ليس** جدار ناري؛ غالباً اسم المضيف غير مضاف أو **CSRF** يحتاج عنوان المتصفح الحقيقي.

عدّل `~/cafe-pos/.env` (استبدل `IP` و `8000` بقيمك):

```env
ALLOWED_HOSTS=localhost,127.0.0.1,IP,*
CSRF_TRUSTED_ORIGINS=http://IP:8000
```

ثم:

```bash
sudo systemctl restart cafe-pos
```

التثبيت الجديد من السكريبت يسأل عن **عنوان الوصول من المتصفح** ويملأ هذين السطرين تلقائياً.

## إدارة الخدمة

```bash
sudo systemctl status cafe-pos      # حالة الخدمة
sudo systemctl restart cafe-pos     # إعادة تشغيل
sudo systemctl stop cafe-pos        # إيقاف
sudo journalctl -u cafe-pos -f      # سجل الأخطاء
```

## التحديث بأمر واحد

عند توفر تحديثات جديدة:

```bash
cd ~/cafe-pos && bash update.sh
```

يسحب التحديثات + يطبق تغييرات قاعدة البيانات + يعيد تشغيل الخدمة تلقائياً.

## أهم الميزات

| الميزة | الوصف |
|--------|-------|
| **نقطة البيع** | شاشة كاشير سريعة — طاولات، توصيل، استلام |
| **إدارة الطاولات** | فتح/إغلاق جلسات، عملاء، مدفوعات جزئية |
| **المنتجات** | جاهزة، مصنّعة (وصفات)، خدمات، عمولة |
| **المواد الخام** | إدارة منفصلة + تصنيع تلقائي |
| **المخزون** | تتبع الكميات، متوسط التكلفة، جرد فعلي |
| **المشتريات** | فواتير شراء + مرتجعات + دفعات موردين |
| **المحاسبة** | قيد يومية مزدوج تلقائي، شجرة حسابات |
| **التقارير** | مبيعات يومية، أسبوعية، تدفق نقدي، حركة أصناف |
| **الموظفين** | رواتب، سلف، مشتريات من المحل |
| **العملاء** | أرصدة آجلة، كشوفات، مدفوعات |
| **الموردين** | أرصدة، كشوفات، ربط مع العملاء |

- **التصميم**: RTL عربي بالكامل
